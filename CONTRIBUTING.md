# Contributing to UpsideDown

First off, thanks for taking the time to contribute! 🎉

## Code of Conduct
This project and everyone participating in it is governed by the [UpsideDown Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

## How Can I Contribute?

### 🐛 Reporting Bugs
- Check if the bug already exists in [Issues](https://github.com/yourusername/upsidedown/issues)
- Use the bug report template when creating a new issue
- Include as many details as possible: Python version, OS, example code

### 💡 Suggesting Enhancements
- Open an issue with the "enhancement" label
- Explain why this feature would be useful
- Keep it weird — this is a 0.42 project after all! 

### 🔧 Pull Requests
1. Fork the repo
2. Create a new branch (`git checkout -b feature/amazing-idea`)
3. Make your changes
4. Test your changes (if possible)
5. Commit with clear messages
6. Push and open a Pull Request

## Styleguides
- Keep it simple — no dependencies
- Python 3.6+ compatible
- Comments are welcome, especially for the weird parts (like 0.42)
- Use meaningful variable names (no `a`, `b`, `c` unless it's XOR)

## The 0.42 Rule
If you change anything related to the magic exponent, you must use 0.42 or explain why your number is better. This is non-negotiable. 😄

## Questions?
Open an issue with the "question" label.

Thank you for being part of UpsideDown! 🔄
